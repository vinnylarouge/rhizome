# Rhizome Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn loom into Rhizome тАФ a packaged macOS Electron app with provider settings (OpenAI key or local OpenAI-compatible endpoints), discussion cores as data, a session library, and an extension registry shipping Obsidian export + AI search.

**Architecture:** The existing Node server (`src/server.js`) stays the heart; Electron wraps it as a `utilityProcess` and loads `http://127.0.0.1:<port>` in a BrowserWindow (LAN access preserved). Settings live in `RHIZOME_HOME/settings.json` and are hot-reloaded тАФ `llm.js` resolves tierтЖТprovider at call time, so no restart is needed. Cores/sessions/extensions are data under `RHIZOME_HOME`.

**Tech Stack:** Node тЙе18 ESM, `ws` (only runtime dep), vanilla JS + D3 frontend, `node --test`, Electron + electron-builder (dev deps).

**Spec:** `docs/superpowers/specs/2026-06-11-rhizome-electron-app-design.md`
**Deviation from spec (improvement):** settings changes hot-reload rather than restarting the server process; strictly simpler and the live board never blinks.
