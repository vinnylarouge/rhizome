> **To control a system you must first make it legible — and the simplification you impose for control destroys the local intelligence that made it work.**

## When it fires
- A reform redraws reality into clean categories (standard plots, grid streets, unified metrics) so the centre can manage it.
- The map the institution acts on is visibly thinner than the territory.
- Things measured improve while unmeasured things you actually cared about quietly rot.

## Questions it forces
- What did this system have to erase about itself to become measurable?
- Whose tacit competence stops mattering once the dashboard exists?

## What to do
Before standardising, inventory the local knowledge the standard will overwrite. Keep a channel where reality can talk back: shadow the metric with a qualitative audit, and let operators override the schema when it lies. Simplify the interface, not the underlying practice.

## Failure mode
Some legibility is genuinely necessary — you cannot run vaccination or taxation on pure metis. Romanticising illegibility blocks coordination that saves lives.

## Out-of-domain example
A benchmark suite makes model quality legible, so labs optimise the benchmark; real-world capability (the illegible thing) stagnates while leaderboard numbers climb.
